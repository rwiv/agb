# Task 1.2: 프론트매터 순서 보존 검증 테스트

## 개요
`Cargo.toml`에 `preserve_order` 피처를 추가한 후, 실제 코드베이스에서 `build` 및 `sync` 동작을 거칠 때 프론트매터의 키 순서가 정상적으로 보존되는지 검증할 수 있는 테스트 환경을 구성합니다.

## 작업 내용
1. `tests/` 디렉터리 내 기존 테스트 코드 또는 신규 테스트 파일을 통해 테스트 시나리오를 작성한다.
2. 프론트매터 키 순서가 불규칙적으로 섞여있는 마크다운 파일(예: `b`, `a`, `c` 순서)을 소스로 사용한다.
3. `Transformer`를 통한 빌드 및 `syncer::patcher::MdPatcher`를 통한 동기화 시뮬레이션을 수행한다.
4. 결과물로 생성되거나 동기화 패치된 마크다운 파일의 프론트매터 키 순서가 원본(예: `b`, `a`, `c`)과 일치하는지 어서션(Assertion)한다.
5. 테스트 코드를 실행하여 정상적으로 통과하는지 확인한다.

## 갱신 대상 파일
- 관련 테스트 코드 (예: `tests/e2e_sync_test.rs` 또는 `src/syncer/patcher.rs` 하단의 단위 테스트 등)

## 성공 기준
* 직렬화 및 역직렬화 과정 이후에도 원본 마크다운 파일의 프론트매터 키 순서가 그대로 유지됨을 검증하는 테스트가 존재해야 한다.
* 모든 테스트(`cargo test`)가 성공적으로 통과해야 한다.
