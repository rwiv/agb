# TASK-2-3: 에러 핸들링 강화 및 unwrap() 제거

## 목적
`syncer` 모듈 내의 잠재적인 패닉 포인트를 제거하고, `anyhow`를 활용하여 사용자에게 보다 친절하고 구체적인 에러 메시지를 제공합니다.

## 작업 내용

1. **unwrap() 제거**
   - `src/syncer/mod.rs`, `skill.rs`, `planner.rs` 등에 존재하는 `.unwrap()`, `.expect()` 호출을 `context()`가 포함된 `Result` 반환으로 교체.
   - 특히 파일 경로나 부모 디렉터리 접근 시 발생할 수 있는 오류 처리 강화.

2. **에러 컨텍스트 추가**
   - 리소스 동기화 실패 시 어떤 리소스(유형, 이름)에서 오류가 발생했는지 명시적으로 포함.
   - 파일 읽기/쓰기 실패 시 대상 경로 정보를 에러 메시지에 포함.

## 검증 방법
- 고의적으로 잘못된 파일 경로 등을 설정하여 에러 메시지가 정확하게 출력되는지 확인.
- `cargo clippy`를 통해 잠재적인 위험 요소 점검.
