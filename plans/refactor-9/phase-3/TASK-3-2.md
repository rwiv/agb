# Task 3-2: 기존 코드 제거 및 최종 검증

## 1. 개요
리팩토링이 완료된 후 불필요해진 기존 코드를 제거하고 전체 시스템의 정합성을 확인합니다.

## 2. 작업 상세

### 2.1. 파일 삭제
- `src/syncer/diff.rs` 파일을 삭제합니다.

### 2.2. 모듈 선언 정리
- `src/syncer/mod.rs`에서 `pub mod diff;` 선언이 새로운 `src/syncer/diff/mod.rs`를 가리키도록 확인합니다.

### 2.3. 통합 테스트 및 E2E 테스트
- `tests/e2e_sync_test.rs` 등 모든 동기화 관련 테스트를 실행하여 기능 퇴보(Regression)가 없는지 확인합니다.

## 3. 검증 방법
- `cargo test` 결과 모든 테스트 통과 확인.
- 실제 `agb sync` 명령어를 샘플 리소스에 대해 실행하여 소스 파일이 올바르게 업데이트되는지 확인.
