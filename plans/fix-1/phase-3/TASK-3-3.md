# TASK-3-3: 버그 리포트 기반의 엣지 케이스(Edge Cases) 검증

## 목적
실제 사용 중에 발생할 수 있는 특수한 상황들을 가정하여 동기화 로직의 견고함을 최종 확인합니다.

## 작업 내용

1. **엣지 케이스 시나리오 테스트**
   - 소스 파일이 비어있거나 잘못된 Frontmatter 형식을 가진 경우.
   - 타겟 파일이 수동으로 삭제되어 존재하지 않는 경우 (무시되어야 함).
   - `exclude` 패턴에 해당하는 파일이 타겟에 추가된 경우 (동기화되지 않아야 함).
   - 파일 권한 문제로 쓰기가 불가능한 경우의 에러 메시지 확인.

## 검증 방법
- 수동 테스트 시나리오 수행 및 결과 기록.
- 실패해야 하는 상황에서 적절한 에러 코드가 반환되는지 확인.
