# Product Requirements Document (PRD)

## 1. 프로젝트 개요 (Overview)

`agb`는 여러 플러그인에 분산된 AI 에이전트 리소스(Commands, Agents, Skills)를 중앙에서 통합 관리하고, 각 타겟 에이전트(Gemini-cli, Claude-code 등)의 규격에 맞춰 자동으로 변환 및 빌드하는 CLI 도구입니다. 프로젝트 로컬에서의 변경사항을 원본 리소스로 안전하게 환원하는 양방향 동기화 기능을 핵심 가치로 제공합니다.

## 2. 배경 및 목적 (Background)

현재 여러 코딩 에이전트들은 각기 다른 설정 방식과 파일 구조를 가지고 있습니다. 이로 인해 동일한 프롬프트나 스킬을 여러 에이전트에서 사용하려면 중복된 파일을 생성하고 관리해야 하는 번거로움이 발생합니다. 

특히 프로젝트마다 서로 다른 설정 파일이 필요하고, 이들이 프로젝트별로 산발적으로 수정될 경우 이를 원본과 동기화하는 것은 매우 어렵습니다. `agb`는 설정 베이스 디렉토리를 통해 리소스를 중앙 관리하고, 프로젝트 진행 중 발생한 변경사항을 다시 베이스로 환원하는 기능을 통해 이 문제를 해결합니다.

## 3. 핵심 가치 (Core Values)

- **중복 제로 (Zero Redundancy):** 단일 마크다운 소스로 여러 에이전트 지원.
- **중앙 집중식 관리 및 유연한 로컬 동기화:** 베이스 디렉토리에서 리소스를 관리하되, 프로젝트 로컬에서의 변경사항을 안전하게 동기화.
- **명시적 관리:** 빌드 설정 파일을 통한 포함 리소스의 엄격한 제어.
- **이식성:** 플러그인 구조를 통한 워크플로우의 쉬운 공유 및 재사용.

## 4. 사용자 타겟 (Target Audience)

- 여러 종류의 AI 코딩 에이전트를 병행하여 사용하는 개발자.
- 복잡한 프롬프트 엔지니어링 결과물(Skills, Sub-agents)을 체계적으로 관리하고자 하는 사용자.

## 5. 주요 요구사항 (Key Requirements)

- **단일 소스 관리**: 모든 에이전트용 리소스를 하나의 저장소에서 관리할 수 있어야 합니다.
- **플러그인 기반 확장성**: 기능을 독립적인 플러그인 단위로 구성하여 재사용성을 높여야 합니다.
- **유연한 설정 오버라이트**: 특정 에이전트에서만 필요한 설정을 원본 소스 훼손 없이 추가할 수 있어야 합니다.
- **자동 포맷 변환**: 각 에이전트가 요구하는 파일 포맷(TOML, Markdown FM 등)으로 자동 변환되어야 합니다.
- **안전한 빌드**: 리소스 이름 충돌이나 금지된 파일 포함 등을 사전에 감지하여 빌드 오류를 방지해야 합니다.

## 6. 성공 지표 (Success Metrics)

- 여러 에이전트 설정을 한 곳에서 관리하여 설정 파일의 중복 발생 건수를 0으로 유지함.
- 동일한 리소스를 서로 다른 에이전트 환경에서 성공적으로 로드하고 작동시킴.
